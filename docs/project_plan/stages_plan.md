# 计划书

## 第一阶段：项目初始化与设计（1-2天）

**核心目标：** 明确数据结构，搭建代码仓库，完成"纸上谈兵"。

1. **Git 仓库初始化**

* **要求：** 任务书强调 Git 提交记录要有意义，不要最后一次性提交 。

* **操作：** 在 GitHub 或 微信开发者代码管理平台创建仓库。建议创建两个文件夹或分支：`client` (前端) 和 `server` (后端)，并配置好 `.gitignore`。

2. **数据库设计 (关键)**

* 根据文档要求 ，你需要设计以下核心数据模型（建议使用 MongoDB 或 MySQL）：

* **User/Admin 表：** 包含账号、密码、角色（商户/管理员）。

* **Hotel 表：** 必须包含酒店名、地址、星级、房型、价格、开业时间 ；状态字段（审核中/通过/拒绝/下线）。

* **Booking/Orders 表：** 虽文档未详细展开预订后逻辑，但为了完整性，建议设计一个简单的订单表关联用户和酒店。

3. **接口定义 (API Design)**

* 规划好 RESTful 接口，例如：
* `POST /api/auth/login` (登录，自动判断角色 )
* `POST /api/hotels` (商户录入)
* `GET /api/hotels` (移动端列表查询，支持筛选)
* `PUT /api/hotels/:id/audit` (管理员审核)

---

## 第二阶段：后端开发 - Node.js (2-3天)

**核心目标：** 提供数据支撑，确保前端有数据可调。

1. **基础框架搭建：** 使用 Express 或 Koa 搭建 Node.js 服务。
2. **鉴权模块：** 实现登录/注册功能。注意：注册页需选择角色，但登录页需自动根据账号判断角色 。

3. **酒店管理接口：**

* 实现 CRUD（增删改查）。
* **重点：** 确保商户修改数据后，数据能实时更新，供移动端调用 。

**Mock 数据填充：** 为了方便后续前端开发，先往数据库里填入一些测试数据（如：上海陆家嘴禧玥酒店 ）。

---

## 第三阶段：PC端管理系统开发 (3-4天)

**核心目标：** 完成数据的录入和审核，为移动端生产内容。

**技术建议：** React + Ant Design (允许使用开源组件库 )。

1. **登录/注册页：** 实现双角色逻辑（商户/管理员）。
   2.商户功能 - 信息录入/编辑 ：

* 使用表单组件，包含必填项（酒店名、价格等）。
* 实现图片上传（可选功能，文档截图中有图片展示 ）。

1.管理员功能 - 审核列表 ：

* 展示所有提交的酒店。
* 操作按钮：通过、不通过（需填写原因）、下线（非物理删除，可恢复）。

---

## 第四阶段：移动端（用户侧）开发 (4-5天)

**核心目标：** 优化用户体验，完成核心查询预订流程。
**技术建议：** Taro (React语法，可编译为小程序) 或 原生小程序。

1.首页 (酒店查询) ：

* **顶部 Banner：** 轮播图，点击跳转详情。
* **核心功能：** 定位（调用小程序 API）、关键字搜索框、**日期选择日历组件**（这是明确要求的得分点 ）。

1.酒店列表页 ：

* **筛选头：** 城市、日期、排序。
* **上滑加载：** **技术难点**。必须实现分页加载或"上滑自动加载"功能，这涉及到"技术复杂度"评分 。

* **性能优化：** 如果数据量大，考虑长列表渲染优化（Virtual List）以获取高分 。

1.酒店详情页 ：

* 展示大图 Banner、基础信息。
* **房型列表：** 必须根据商户端录入的价格**从低到高排序** 。

---

## 第五阶段：优化与加分项冲刺 (2天)

**核心目标：** 对照评分标准  进行优化。

1. **用户体验 (UX)：**

* 检查页面视觉是否美观，布局是否合理。
* 确保在手机上无兼容性问题。

1. **代码质量：**

* 抽取通用组件（如：酒店卡片组件、搜索栏组件）。
* 编写清晰的 `README.md` 说明文档。

1.创新性 (10分) ：

* **建议方向：**
* **技术：** 使用 WebSocket 实现价格变动实时通知，或者使用 TypeScript 保证代码规范。
* **功能：** 实现“地图模式”找房，或者“比价”功能。

---

## 第六阶段：测试与交付 (1天)

1. **联调测试：** PC端修改价格，手机端下拉刷新看价格是否变化（验证“实时更新” ）。

2. **清理代码：** 删除无用的 console.log。
3. **最终提交：** 确保 Git 仓库完整，包含说明文档。
